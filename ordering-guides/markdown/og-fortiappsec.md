How It Works:

The FortiAppSec Cloud delivers multi-service capabilities by combining application

security, service optimization, and threat analytics into a unified interface:

‚ÄĘ Web Application Firewall: AI/ML-based Web Application Firewall (WAF) that provides

real-time protection against the OWASP Top 10 and zero-day attacks targeting web

applications and APIs. It also includes Client-Side protection and Account Takeover

protection.

‚ÄĘ DDoS Protection: Secure all assets and ensure uptime with cloud-based Layer 3-7

DDoS protection.

‚ÄĘ Application Delivery (CDN, GSLB): Enhance application performance with full CDN

capabilities and Global Server Load Balancing (GSLB) to optimize traffic and user

experiences and ensure availability across data centers.

‚ÄĘ Advanced Bot Protection: Utilizes machine learning and behavior analysis to detect

and mitigate malicious bot activities, protecting against automated attacks like account

takeover, scraping, and fraud.

‚ÄĘ Client-Side Security: The Client-Side Protection service provides a unified interface

for detecting, preventing, and managing browser-based threats and supporting PCI

DSS 4.0 client-side security requirements.

‚ÄĘ Threat Analytics: Provides comprehensive visibility with real-time monitoring, enabling

insights into traffic patterns, anomaly detection, and detailed attack analysis. Using

ML to identify patterns it helps speed up alert security investigation and helps SOC

analysts focus on the important threats.

FortiAppSec Cloud FAQ

How are the FortiAppSec Cloud SKUs structured?

The new price list moves from the previous stackable model to a seat-based model. Customers choose the number of seats that
represent the overall number of web applications or bandwidth required.

How are SKUs structured?

There are three types of SKUs:

1. Cloud WAF SKUs (previously FortiWeb Cloud):

a. Customers must buy a bandwidth SKU and a Web application SKU.

b. Customers choose between Standard, Advanced and Enterprise SKUs. If they purchase a Enterprise bandwidth SKU, they

must also purchase the Enterprise web application SKU. You cannot mix and match between the plans.

2. Standalone service GSLB: You are not required to purchase Cloud WAF SKUs for the Standalone SKUs

3. Add-on SKUs: These SKUs also require that you buy Cloud WAF SKUs. Available SKU - SOCaaS.

Are these stackable SKUs?

No. These are Seat-based SKUs. You choose the amount of the SKU needed instead of the stackable model where you buy
multiple of a certain SKU.

How does the Cloud WAF pricing work?

Customers purchase bandwidth and the number of web applications for which they need protection. They must choose the plan
type (Standard, Advanced or Enterprise, must be selected for bandwidth and apps, no mix and match).

Can customers buy Cloud WAF features A-La-Carte?

No. Customers can only choose between the Standard, Advanced or Enterprise plans. No A-La-Carte feature choosing.

Can customers choose the Standard plan for certain applications and Advanced plans for other applications?

No. Customers can NOT mix and match the plan between applications. An account can only have one type of plan Standard,
Advanced or Enterprise. For example, if a customer buys 20 applications all of them must be Standard, Advanced or Enterprise.
This is true for both the Web apps SKU and Bandwidth SKU.

Can customers choose the Standard plan for web apps and Advanced plan for bandwidth?

No. Only one type of plan can be chosen. Customers must choose the same plan for Web apps SKU and Bandwidth SKU.

Is GSLB add-on SKUs or Standalone?

GSLB is standalone SKUs. You are not required to purchase Cloud WAF SKUs for these services.

Can I buy SOCaaS SKUs without Cloud WAF SKUs?

SOCaaS is a service provided for Cloud WAF customers and an Add-on SKU. You must purchase Cloud WAFs SKUs first.

Is the Threat Analytics service included in the Standard plan?

No. Threat Analytics is only included in the Advanced plan.

Can customers upgrade from one plan to another?

Yes. Customers can upgrade to higher plans. Use the co-term tool for this.

Can I purchase 2 year or 4 year contracts? I don‚Äôt see this option on the price list

If you need a contract length that‚Äôs not included in the price list simply purchase multiple of the SKU and register them right
after the other. For example, for a two year contract for 5-24 applications purchase 2XFC2-10-UCAPF-1116-02-12 and register
consecutively. Please follow registration rules and make sure contracts are registered within the 12 months after purchase.

Can I purchase and use FortiDAST as a standalone solution?

No, FortiDAST is now only available within FortiAppSec Advanced bundles. DAST can be used exclusively on applications
protected by FortiAppSec; standalone DAST is no longer offered.

Renewals and Migration from FortiWeb Cloud Contracts

What are Legacy contracts?

When FortiAppSec was introduced all FortiWeb Cloud, FortiGSLB and FortiABP contracts were migrated to FortiAppSec and are
called ‚ÄúLegacy contacts‚ÄĚ. Functionality wise nothing changed and the same capabilities are now provided via FortiAppSec.

How do customers migrate from FortiWeb Cloud to FortiAppSec?

Customers with FortiWeb Cloud contracts were migrated to FortiAppSec and their contacts are now called ‚ÄúLegacy contract‚ÄĚ.
Upon renewal customers can simply purchase FortiAppSec contracts and register them to start once the FortiWeb Cloud
contracts expire.

Can I use both FortiWeb Cloud and FortiAppSec contracts under the same account?

No. Only one license type can be enabled for a certain account. If you registered multiple contracts enable the right one in
FortiAppSec‚Äôs Contracts tab.

Can I add additional applications or bandwidth to my existing FortiAppSec contract?

Yes. Use the co-term tool to add additional bandwidth/applications seats.

Can I add additional applications or bandwidth to my existing FortiWeb Cloud contract (also called ‚ÄúLegacy contract‚ÄĚ)?

Yes. You can buy additional bandwidth/application contracts and stack them.

Can I add extend my FortiWeb Cloud contract (also called ‚ÄúLegacy contract‚ÄĚ) to a different date?

No. FortiWeb Cloud contracts are EOL‚Äôd and you cannot extend them. While you can purchase additional bandwidth/applications
throughout the contract term, you cannot extend the date further. You must move to FortiAppSec contracts
